# Runtime Contract

更新时间：2026-05-20

这份文档只描述运行期边界，不讨论产品路线。
目标是把安装、更新、运行、导入导出这些入口的职责固定下来，减少后续继续散开。

## 1. 用户入口

### 安装

- 入口：`scripts/install.sh`
- 负责：
  - 检查 Python / git / 平台包管理器前置条件
  - 拉取项目代码到用户目录
  - 建立 `.venv`
  - 安装运行时依赖
  - 写入启动命令 `zaomeng`
  - 给出后续命令提示
- 不负责：
  - 修改用户业务数据
  - 覆盖现有小说运行数据
  - 覆盖用户本地模型配置

### 更新

- 入口：
  - CLI: `zaomeng update`
  - Web UI: 首次启动弱提示更新
- 负责：
  - 对比本地版本与远端版本
  - 仅在版本不一致时执行更新
  - 保留用户数据目录、配置目录、已导入小说与运行记录
  - 更新完成后提示重新加载或自动刷新
- 不负责：
  - 重置用户配置
  - 删除用户创建的角色卡 / 场景卡 / 内置导入资产

### 运行

- 入口：
  - CLI: `zaomeng`
  - CLI: `zaomeng web --reload`
  - 开发入口：`scripts/run_webui.py`
- 负责：
  - 启动 Web UI 服务
  - 检查基础目录是否存在
  - 加载模型配置、小说运行目录、内置小说目录
  - 首次启动时做轻量更新检查
- 不负责：
  - 自动重建用户运行数据
  - 静默吞掉关键启动错误

## 2. 数据目录边界

### 安装根目录

- 典型位置：`$HOME/.local/share/zaomeng`
- 这里放：
  - 项目代码副本
  - `.venv`
  - 启动脚本
- 更新时允许覆盖。

### 用户数据目录

- 典型位置：
  - Web UI：`.zaomeng-web`
  - 其他平台按当前运行配置解析
- 这里放：
  - 运行中的小说 `runs`
  - 导入的小说包
  - 内置小说索引缓存
  - 会话数据
  - 本地卡片数据
  - 模型设置
- 更新时必须保留。

### 运行目录

- 单本小说一次蒸馏 / 导入后的产物目录
- 允许包含：
  - `run_manifest.json`
  - 人物档案
  - 关系图谱
  - payload / artifact 索引
  - 会话目录
- 运行目录是资产容器，不是安装容器。

## 3. 导入 / 导出边界

### 导出小说包

- 负责：
  - 从当前 run 提取可复用资产
  - 写出稳定 package manifest
  - 保留必要人物档案、图谱、元数据
  - 缺少可选产物时降级导出，不应硬失败
- 不负责：
  - 导出运行中的临时 UI 状态
  - 导出无意义的本地绝对路径

### 导入小说包

- 负责：
  - 校验 package manifest
  - 兼容旧字段、旧路径、Windows 路径残留
  - 缺少 graph / payload 时按降级模式导入
  - 导入后生成可运行的 run manifest
- 不负责：
  - 强求所有旧包都完整无损
  - 遇到可降级缺失时直接拒绝导入

## 4. Web UI 运行反馈要求

以下流程都应尽量统一四件事：

- loading：当前正在做什么
- impact：失败是否影响聊天主流程
- next step：成功或失败后下一步做什么
- recoverability：是否可以重试 / 继续

当前重点流程：

- 创建小说运行
- 继续蒸馏
- 关系图生成
- 导入小说包
- 导出小说包
- 克隆内置小说
- 检查更新 / 执行更新

## 5. 兼容原则

- 新逻辑优先读取 canonical 字段
- 旧字段只在 compatibility layer 里处理
- Web UI 与 skill 不各写一套兼容修补
- manifest 兼容、路径兼容、包兼容应尽量集中

## 6. 禁止事项

- 更新流程不得覆盖用户配置和用户资产
- 导出包不得写入宿主机器绝对路径作为 source-of-truth
- 运行入口不得把安装期逻辑和业务运行逻辑混在一起
- compatibility 修补不得继续散落到 UI、列表、导入、视图多个模块里重复出现

## 7. 代码目录职责映射（P2-3）

为了避免“边界写在文档里、代码里却继续散开”，运行期主线职责固定到以下目录：

- 安装入口与安装期脚本：
  - `scripts/install.sh`
  - 仅负责安装期依赖与启动命令落地，不写业务运行数据
- 运行入口：
  - `scripts/run_webui.py`
  - `src/cli`
  - 仅负责启动与参数分发，不承担导入导出业务逻辑
- 更新与版本检查：
  - `src/web/service_facades/update.py`
  - `scripts/release_skill.py`
  - 保持“更新程序不覆盖用户业务数据”约束
- 运行编排与状态推进：
  - `src/web/service_facades`
  - `src/web/pipeline`
  - `src/web/run_ops`
  - 负责 run 创建、推进、停止、重启与状态投影
- manifest 与兼容层：
  - `src/web/manifest`
  - 兼容修补集中入口（路径重写、导入语义迁移、缺失产物降级）
- 导入导出小说包：
  - `src/web/run_ops/packages.py`
  - 负责 package manifest 校验、schema 兼容、导入导出边界
- 对话运行态：
  - `src/web/chat`
  - canonical `session state` 的读写与派生视图
- 前端展示层：
  - `src/web/static`
  - 只消费 canonical/derived 数据，不新增后端兼容补丁

约束：

- 新增代码若涉及边界变更，必须同时更新本节映射与对应测试断言
- 目录职责冲突时，优先保持“兼容层集中、入口轻量、业务层单一职责”
